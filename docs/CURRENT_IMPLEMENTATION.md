# Current Implementation Status

This document is the maintained source of truth for AllEnricher v2.1.1. Historical planning notes and generated audit outputs are intentionally excluded from the main repository.

## Product Surfaces

- CLI: `allenricher analyze`, `build`, `download`, `serve`, `list`, `config`, `check-update`, `cleanup`, `list-versions`, `list-species`, `query-species`, and `tf-enrich`.
- Python package: enrichment, database loading, plotting, report generation, AI interpretation, and configuration modules.
- REST API and local Web workbench: implemented in `allenricher/api/server.py` and `allenricher/api/static/index.html`.
- HTML reports: generated by `allenricher/report/generator.py`.

## Supported Analysis Methods

| Method | Implementation | Required input | Default gene-set size |
| --- | --- | --- | --- |
| ORA / `hypergeometric` | one-sided hypergeometric test | query gene list | 3 to unbounded |
| GSEA / `gsea` | Bioconductor `fgseaMultilevel` bridge | ranked genes with signed numeric weights | 15 to 500 |
| TF GSEA | Bioconductor `fgseaMultilevel` bridge | ranked genes with signed numeric weights | 15 to 5,000 |
| ssGSEA / `ssgsea` | Bioconductor `GSVA` bridge | gene-by-sample expression matrix | 1 to unbounded |
| GSVA / `gsva` | Bioconductor `GSVA` bridge | gene-by-sample expression matrix | 1 to unbounded |

Size filters are applied after intersecting gene sets with the method-specific gene universe.

## Supported Databases

| Database | Scope | Notes |
| --- | --- | --- |
| GO | registry-defined species | functional ontology annotations |
| KEGG | KEGG organisms | pathway gene sets |
| Reactome | Reactome-supported species | pathway gene sets |
| WikiPathways | WikiPathways-supported species | pathway gene sets |
| DO | human | disease ontology associations |
| DisGeNET (`v20190612`) | human | retained free snapshot |
| TRRUST v2 | human and mouse | TF-target interactions |
| ChEA3 | human | TF-target gene-set libraries |
| AnimalTFDB | AnimalTFDB species | TF annotations with prepared target sets |
| hTFtarget | human | tissue-specific TF-target associations |
| CUSTOM | user-defined | built from user annotations or GMT-style gene sets |

`MSigDB` is not a supported public database in v2.1.1.

## Current Figure Types

R is the canonical renderer for GSEA, ssGSEA, and GSVA publication figures. Python plotting remains as a minimal fallback for those methods, plus workflow helpers such as running-ES table generation, report fallback figures, contact sheets, and visual audits. ORA currently uses the maintained Python bar/lollipop renderer.

| Method | Canonical figures | Fallback / helper scope |
| --- | --- | --- |
| ORA | Python `barplot`, `lollipop` | no duplicate R ORA workflow is exposed |
| GSEA | R `enrichment`, `enrichment2`, `barplot`, `lollipop`, `ridgeplot`, `emapplot` | Python `enrichment`, `enrichment2`, `barplot`, `lollipop`, `ridgeplot` when `--python-plots` is used or R is unavailable |
| ssGSEA / GSVA | R `heatmap`, `group_comparison`, `correlation` | Python `heatmap`, `group_comparison`, `correlation` when `--python-plots` is used or R is unavailable |

Removed or non-current general figure names include `bubble`, `dotplot`, `network`, `upset`, `volcano`, `method_comparison`, `cnetplot`, and `circos`. They are not part of the maintained plotting surface.

## AI Interpretation

- AI input evidence is generated by code before any model call.
- Supported modes are `summary`, `reviewer`, and `caption`.
- Default evidence counts are ORA top 15, GSEA top 10, and ssGSEA/GSVA top 10 by group difference.
- Every data-linked claim must cite a valid `evidence_id`.
- AI failure is reported separately and must not make a completed analysis fail.
- The Web workbench exposes AI interpretation through the existing backend configuration and API path.

## Web and Report Behavior

- The Web workbench uses a method-first workflow and shows unsupported databases as disabled, not hidden.
- The Web workbench uses the default R-backed figure path for GSEA, ssGSEA, and GSVA; it does not expose an R/Python plotting switch. The Python fallback is CLI-only through `--python-plots`.
- Species identity is TaxID-first; KEGG codes and names are aliases.
- Database support counts include TF databases and fixed human-only or human/mouse databases.
- Reports include generated figures, result tables, run metadata, AI interpretation state, and a Materials and Methods writing reference when metadata is available.
- Missing versions or references must be displayed as `To be added`, not guessed.

## Reproducible Container

- The root `Dockerfile` pins the Bioconductor 3.23 base image by immutable digest and verifies R 4.6.1, `fgsea` 1.38.0, and `GSVA` 2.6.2 at build time.
- Direct Python runtime requirements are constrained, and the complete resolved Python package list is recorded inside the built image.
- Inputs and database snapshots are mounted at `/work`; they are not embedded in the image.
- The image runs as the non-root `rstudio` user and exposes the same CLI, Web, and REST code paths as a local installation.
- The optional R-only `emapplot` renderer is excluded from the core image because `aPEAR` and one of its dependencies are archived upstream.
